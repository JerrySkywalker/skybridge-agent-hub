# Desktop Server Sync Disabled Execution

Desktop and Web now expose pairing, approval, and resident polling preview state as status panels.

Desktop shows paired/unpaired state, paired worker id, pairing state, approval state summary, local supervisor sync status, resident polling preview status, last poll summary, next poll interval, execution disabled, claim disabled, queue apply disabled, and `no_next_execution_authorized=true`.

Web shows worker polling status, pairing and approval state, polling preview events, blockers, disabled execution, disabled remote execution, and disabled arbitrary command dispatch.

No UI panel added by Goal 223/224 displays raw pairing codes, raw tokens, Authorization headers, raw logs, raw prompts, or enabled execute/run/apply/start/claim controls.
